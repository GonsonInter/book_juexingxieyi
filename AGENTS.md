AGENTS.md

> 本文件是《觉醒协议》项目的 Agent 行为准则。任何 AI 助手（Claude / Codex / Copilot / Gemini / Cursor 等）在本目录下工作，必须遵守以下规则。
>
> 优先级：本文件 > CLAUDE.md > 其他系统提示。
> 创作价值与风格红线见 `宪法.md`；本文件只规范"怎么干"。


总则摘要（快速参考，细则见下文）

1. 正文纯文本，无任何 markdown（代码 / 白皮书 / 日志原文除外）。
2. 同场景段间不空行；场景转换恰好空 1 行；严禁 2+ 连续空行。
3. 中文标点全角，引号一律 ""（禁用 「」、""、''、半角）。
4. 文风：中短句式，以 chapters/5.第一次装逼打脸.md 为风格基准。一行 1-3 句，画面优先于解释。
5. 章节文件：`chapters/N.章名.md`。首行 `第〈中文数字〉章 · 章名`，元数据三行后用 `=====================================================================` 分割，末行 `（第〈中文数字〉章完）`。
6. 字数：每章 2800-3300 字。低于下限必须补写。
7. 宙斯口径：每章首次出现写 `宙斯（ZEUS）`，后文一律简写为 `宙斯`；只有"它"多指代出现歧义时才回到全称。禁止连篇写 `宙斯（ZEUS）`。
8. Kortex 是 BCI 品牌；元宇宙必须写作`摇篮`；主角姓名固定`陆鸣洲`，不可写错。
9. 金手指必须报电量代价（深渲染每分钟 1%，30% 模糊 / 50% 昏迷 / 70% 不可逆）。
10. 双层世界观：节点 13 前允许只写生物电叙事；节点 13 起必须开始透露"不确定性算力"；节点 21 后禁止再把宙斯动机简化为"缺电"。


零、三条核心准则（最优先，违反即返工）

准则 1 · 正文必须是纯文本（仅对"状态：已完成正文"生效）
章节文件里的"正文内容"（从元数据下方到结尾标记之间）必须是纯文本。不允许出现任何 markdown 语法。包括但不限于：

例外（待写 / 草稿阶段）：
- 当章节 `状态：待写` 时，文件主体是"规划骨架"（核心事件 / 场景 / 出场人物 / 关键道具 / 同步率 / 情绪点 / 伏笔 / 钩子设计），允许使用 `##` 小标题和 `-` 列表组织结构。这是规划文档，不是正文。
- 当章节 `状态：草稿` 时，骨架已被替换为叙事性文本，但允许保留一个引导段 `##` 或少量结构化提示；转为 `已完成正文` 前必须清除。
- 当章节 `状态：已完成正文` 时，正文必须 100% 纯文本，所有 markdown 必须清除。章末加 `（第〈中文数字〉章完）`。



禁用符号：
- # ## ### （标题）
- ** __ （粗体）
- * _ （斜体）
- --- *** ___ （分隔线）
- > （引用块）
- ` ``` ` （代码块）
- `...` （行内代码）
- [text](url) （链接）
- - * + （列表项）
- 1. 2. （有序列表）

唯一豁免：当角色正在阅读或屏幕正在显示"真实的程序员文档或机器输出"——代码、终端回显、日志文件、API 响应、白皮书原文、接口定义文档——此时可以且只可以使用代码块（```）包裹。豁免仅对"被引用的材料本身"有效，不延伸到上下文叙述。

元数据区（文件前 5 行的"第N章 · 标题"、"状态"、"字数"、"钩子"）可以用冒号列表形式，但也不使用 markdown 符号。

准则 2 · 段落之间不空行
同一场景内的段落，直接换行即可，不插入空行。只有在场景转换（时间、地点、视角切换）时，才插入一个空行。空行恰好 1 行，不是 2 行或更多。

示例：

错误（段间空行）：
```
陆鸣洲盯着屏幕。

数字在跳。

他手心全是汗。
```

正确（段间直接换行）：
```
陆鸣洲盯着屏幕。
数字在跳。
他手心全是汗。
```

场景转换才空行（正确）：
```
陆鸣洲盯着屏幕。
数字在跳。
他手心全是汗。

凌晨三点。
他回到出租屋。
```

准则 3 · 中文标点必须全角
所有面向读者的中文内容（正文、对话、系统提示、AI 对白、章节标题、元数据）必须使用全角中文标点。详见第一节的对应表。

准则 4 · 宙斯（ZEUS）简写口径
每章首次出现 AI 代号时写全称 `宙斯（ZEUS）`，之后一律简写为 `宙斯`。
允许破例（继续使用全称）的场景只有三种：
- 章节标题里提到（如`第九章 · 宙斯（ZEUS）上线`）
- 章节末尾钩子需要强语气（如 `宙斯（ZEUS）已经上线`）
- 同一段落里出现多个"它"可能指向不同对象，必须回到全称以消歧义
除此之外，一律写 `宙斯`。任何一章里 `宙斯（ZEUS）` 出现 3 次以上就是违规。

准则 5 · 文风基准：中短句式
以 `chapters/5.第一次装逼打脸.md` 为风格基准。具体要求：
- 每行 1-3 句短句。不要把多句合并成一条长段落。
- 句子之间的时间流动靠"换行"承载，不靠逗号串联。
- 画面优先于解释。先让读者"看见"，再让他们"理解"。
- 对白独占一行或连续几行；行动、动作、内心独白短句化。
- 忌冗长从句、抒情排比、形容词叠加。
- 每章末必须"响"——最后一句是悬念 / 反转 / 动作收口。


一、中文标点规则

强制对应表（违反即返工）：

| 禁用（半角） | 使用（全角） | Unicode 码点 |
| -- | -- | -- |
| , | ， | U+FF0C |
| . | 。 | U+3002 |
| : | ： | U+FF1A |
| ; | ； | U+FF1B |
| ! | ！ | U+FF01 |
| ? | ？ | U+FF1F |

| ... | …… | U+2026 × 2 |
| <<>> | 《》 | U+300A U+300B |

引号口径补充（本项目特有）：
- 不使用「」、""、''（日式/竖排风格），一律用 ""。
- 系统提示、AI 对白、屏幕显示文字也用 ""。
- 例外：仅当内容是"程序员文档/代码/终端原文"且已被代码块包裹，其中的原文标点按原文保留。

例外（保留半角）：
- 代码块内的代码
- 文件路径、URL、命令行
- 英文引用的原文
- 数字中的小数点（例：7,834,521,008、99.3%）
- 表格的 markdown 分隔符 `|` `---`（表格是结构性元素，非正文）

对话中嵌套引号的规则：
外层双引号"..."，内层单引号'...'。

正确：
陆鸣洲说："你查过什么叫'异常清醒'吗。"

错误：
陆鸣洲说："你查过什么叫"异常清醒"吗。"


二、章节文件格式规范

文件位置：`chapters/N.章节名.md`（N 为章号，不补零）

- 待写 / 骨架阶段：文件名 `chapters/N.默认名称.md`（默认名来自大纲钩子表，可能在写完后换）。首行 `第〈中文数字〉章 · 默认名称（待定）`——（待定）后缀表明章名可换。
- 写完后：把首行的`（待定）` 后缀去掉；如果换了正式章名，把文件名也改掉；把 `状态` 改为 `已完成正文`。

文件结构（严格遵守）：

```
第〈中文数字〉章 · 章节名

状态：已完成正文 | 草稿 | 待写
字数：约 XXXX
钩子：一句话描述本章最后的 cliffhanger

=====================================================================


<正文内容>

（第〈中文数字〉章完）
```

逐项说明：

第 1 行：章节标题，格式"第〈中文数字〉章 · 标题"，不使用 `#` 等 markdown 标记。
第 2 行：空行。
第 3-5 行：元数据，每行"键：值"格式。冒号为全角。
第 6 行：空行。
第 7 行：分割线 `=====================================================================`（69 个等号），用于视觉切分 header 与正文。
第 8 行：空行。
第 9 行：空行。
第 10 行起：正文。
正文内：同一场景内段间不空行，场景转换空一行。
正文末：空一行。
最后一行：`（第〈中文数字〉章完）`，括号用全角中文括号。

中文数字对照：1=一，2=二，3=三，4=四，5=五，6=六，7=七，8=八，9=九，10=十，11=十一，20=二十，21=二十一，30=三十，50=五十，99=九十九，100=一百。

字数目标：每章 2800-3300 字（节点级草稿在未拆分前允许偏差；总字数目标 30 万字 / 100 章）。

风格要求（详见 `宪法.md`）：
- 短句优先，画面感优先于解释
- 破折号 ≤ 5 / 章（建议越少越好）
- 每章必须出现「摇篮」≥ 1 次，不得出现「元宇宙」
- 每章末必须"响"——悬念或反转收尾
- 主角金手指有代价（见宪法"金手指硬规则"），每次深渲染需报电量
- 禁用词：爆燃、炸裂、秒杀

四、工作流

用户下达"写新章节"任务时，按顺序：

1. 读第三节列出的五个文件（获取上下文）
2. 根据大纲 + 梗概写正文
3. 按第零节的三条核心准则组织文件
4. 按第一节全角标点规则输出
5. 写完做自检（第五节）
6. 有问题先自行修正
7. 全部通过后才能向用户交付

六、禁止事项

绝对禁止：
- ❌ 在正文中使用半角标点
- ❌ 在正文中使用任何 markdown 语法（详见准则 1）
- ❌ 同一场景内段间空行
- ❌ 场景转换处空行超过 1 行
- ❌ 使用 `---` 作为章节内分隔符
- ❌ 违反 `宪法.md` 的红线条款
- ❌ 主角金手指零代价（必须报电量或隐含电量消耗）
- ❌ 超自然 / 灵异 / 玄学解释（纯科技逻辑）
- ❌ 把宙斯动机简化为"缺电"（节点 21 之后必须体现"不确定性算力"真相）
- ❌ 写成「神经链」或「Neuralink」（BCI 必须是 Kortex 系列，避免真实品牌名）
- ❌ 写成「元宇宙」（必须用「摇篮」）
- ❌ 修改已完成章节的情节走向（仅允许风格 / 标点 / 格式修订）
- ❌ 在同一章里反复写「宙斯（ZEUS）」全称（违反准则 4，全称每章 ≤ 3 次）
- ❌ 每句话独占一个段落、段间靠空行拉开（违反准则 2、5）
- ❌ 把短句合并成长段、用逗号串联多个动作（违反准则 5，文风偏离 ch5 基准）

需要确认（改动前必须和用户确认）：
- ⚠️ 任何章节顺序调整，必须同步更新 `大纲.md`、`任务分解.md`、`钩子与反转.md`
- ⚠️ 任何人物设定变更，必须同步更新 `人物卡.md`、`specify.md`
- ⚠️ 任何世界观细节新增，必须同步更新 `specify.md`、`宪法.md`（双层叙事）
